Log Analysis and Detection

Learn how to review system and network logs to identify suspicious activity.

View recent system logs

sudo journalctl -n 100

Inspect the latest system events for unusual login or process activity.

Check listening ports

sudo ss -tulpn

Look for unexpected services or network listeners that could indicate compromise.

Monitor authentication events

sudo grep 'Failed password' /var/log/auth.log | tail -n 20

Review failed login attempts to detect brute force or unauthorized access attempts.
