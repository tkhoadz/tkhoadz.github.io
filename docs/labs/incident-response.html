Incident Response Basics

Learn the first steps to take when suspicious activity is detected.

Verify the alert

Check logs, network activity, and system status before taking action.

Collect evidence

sudo dmesg | tail -n 50
sudo journalctl -n 50

Gather recent system messages and logs to understand what happened.

Isolate the system

If you suspect compromise, remove the affected host from the network and preserve its state for analysis.
